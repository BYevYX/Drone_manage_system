# Руководство по разработке

## Режим разработчика для тестирования заказов

Для удобства разработки и тестирования функциональности заказов без необходимости аутентификации создан специальный режим разработчика.

### Настройка режима разработки

1. **Создайте файл `.env.local`** в корне проекта:
```bash
cp .env.local.example .env.local
```

2. **Установите переменные окружения** в `.env.local`:
```env
# Основные настройки API
NEXT_PUBLIC_API_URL=http://51.250.43.77:8080

# Обход аутентификации в режиме разработки
NEXT_PUBLIC_BYPASS_AUTH=true
```

3. **Перезапустите сервер разработки**:
```bash
npm run dev
```

### Доступные возможности

#### 1. Обход аутентификации
При установке `NEXT_PUBLIC_BYPASS_AUTH=true` компонент `ProtectedRoute` будет пропускать проверку аутентификации и показывать желтую полосу с индикатором режима разработки.

#### 2. Тестовая страница заказов
Доступна по адресу: `http://localhost:3000/dev/order-test`

**Возможности тестовой страницы:**
- Тестирование полного мастера создания заказа
- Тестирование формы кадастровых данных
- Тестирование выбора вегетационных индексов
- Все действия логируются в консоль браузера

#### 3. Быстрый доступ из Header
В режиме разработки в Header появляется кнопка "DEV" для быстрого перехода к тестовой странице.

### Компоненты для тестирования

#### ServiceSelectionWizard
Полный мастер создания заказа с шагами:
1. Выбор услуги
2. Выбор полей
3. Выбор дронов
4. Выбор материалов
5. Настройка параметров
6. Вегетационные индексы
7. Кадастровые данные
8. Проверка заказа

#### CadastralPlotForm
Форма для ввода данных участка:
- Выбор способа задания участка (кадастровый номер или параметры)
- Ввод кадастрового номера с получением данных
- Ручной ввод параметров участка (площадь, координаты)
- Ввод данных владельца участка
- Кнопки "Сохранить данные о заказе" и "Сделать заказ"

#### VegetationIndexSelector
Компонент выбора вегетационных индексов:
- 6 различных индексов (NDVI, EVI, SAVI, GNDVI, NDRE, CIG)
- Подробные описания и формулы
- Интерпретация результатов
- Рекомендации по применению

### Функциональность согласно ТЗ

Все требования из технического задания реализованы:

#### ✅ 3.1. Проверка наличия незавершенных заказов
- Страница "Мои заказы" (`/dashboard/orders`)
- Обнаружение черновиков с уведомлениями
- Кнопка "Продолжить оформление заказа"

#### ✅ 3.2. Начать оформление заказа
- Кнопки "Начать заказ" на страницах услуг
- Кнопка "Новый заказ" в панели управления

#### ✅ 3.3. Выберите услугу
Минимальная конфигурация включает выбор из 4-х списков:
1. **Услуги*** (обязательный) - ServiceCatalog
2. **Дроны** - DroneManagement
3. **Дополнительное оборудование** - в мастере заказов
4. **Материалы** - MaterialManagement

#### ✅ 3.4. Обработка с использованием карт вегетационных индексов
- Компонент VegetationIndexSelector
- Выбор из 6 индексов с подробными описаниями
- Номер индекса вносится в таблицу заказов

#### ✅ 3.5. Введите желательные сроки оказания услуги
- Календарь для выбора даты и времени
- Поддержка диапазона дат

#### ✅ 3.6. Введите адрес участка для оказания услуги
- Поля ввода адреса
- Поддержка GPS координат

#### ✅ 3.7. Кадастровый номер участка или параметры участка
- **a) Кадастровый номер** - ввод и получение данных из кадастра
- **b) Параметры участка** - ввод площади и координат точек периметра

#### ✅ 3.8. Данные о владельце участка
- ФИО владельца
- Контактные данные (телефон, email)
- Тип и номер документа
- Адрес владельца

#### ✅ Обязательные кнопки
1. **"Сохранить данные о заказе"** - сохранение черновика
2. **"Сделать заказ"** - финальное оформление

### Отладка и логирование

Все действия в тестовых компонентах логируются в консоль браузера:
- Выбор услуг и параметров
- Отправка форм
- Создание заказов
- Ошибки API

### Безопасность

⚠️ **Важно**: Режим разработчика работает только при `NODE_ENV=development`. В продакшене все проверки аутентификации остаются активными.

### API Integration

Система готова к интеграции с API: `http://51.250.43.77:8080/docs/`

Все компоненты используют типизированные API вызовы через:
- `ordersApi.createOrder()`
- `fieldsApi.getFields()`
- `dronesApi.getDrones()`
- `materialsApi.getMaterials()`

### Запуск тестирования

1. Установите `NEXT_PUBLIC_BYPASS_AUTH=true` в `.env.local`
2. Запустите `npm run dev`
3. Перейдите на `http://localhost:3000/dev/order-test`
4. Тестируйте все компоненты функциональности заказов
5. Проверяйте логи в консоли браузера

Система полностью готова к тестированию и соответствует всем требованиям технического задания.
